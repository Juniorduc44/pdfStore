# PDF Store App

Anonymous PDF storefront with LNURL-pay checkout and time-limited downloads.

## Structure

- `app/` — Next.js routes, storefront pages, and API endpoints
- `components/` — UI components for catalog and checkout
- `data/` — JSON catalog generated by the import workflow
- `storage/` — private paid PDFs, not publicly served
- `public/uploads/covers/` — product covers
- `public/uploads/previews/` — preview PDFs exposed to browsers
- `scripts/` — prompt-driven product import tooling
- `docs/` — app-specific architecture notes

## Hosting Model

- App: Vercel recommended, since pages and API routes stay on one domain
- Payment backend: OpenNode via API

## Core Features

- anonymous storefront browsing
- prompt-driven PDF import with metadata capture
- price capture on every upload with a chosen currency code
- automatic catalog updates after import
- live or draft product status
- featured products, tags, author, page count, and file size metadata
- preview PDFs per title
- OpenNode-powered paywall checkout with Lightning invoice creation
- browser-side paywall with invoice generation, polling, and manual unlock check
- short-lived tokenized download links after payment
- live display currency switcher with BTC, SATS, USD, and custom ISO currency codes

## Importing A New PDF

Run:

```bash
npm run add-product
```

The script prompts for:

- title
- slug
- short description
- detailed product copy
- author
- pricing currency code
- price amount in that currency
- tags
- page count
- featured flag
- preview enabled
- PDF path
- cover image path
- preview PDF path when preview is enabled
- live or draft status

After import:

- the paid PDF is copied into `storage/pdfs/`
- the cover is copied into `public/uploads/covers/`
- the preview PDF is copied into `public/uploads/previews/`
- `data/products.json` is updated automatically

Preview policy: the preview must be a separate excerpt file. The importer will
reject using the full paid PDF as the preview.

## Live FX Display

The storefront has a global currency selector in the header. It supports:

- `SATS`
- `BTC`
- common fiat currencies such as `USD`, `EUR`, `GBP`, `CAD`, `AUD`, `JPY`
- custom ISO codes supported by the live rate feed

Rates are fetched from:

- `https://cdn.jsdelivr.net/npm/@fawazahmed0/currency-api@latest/v1/currencies/btc.json`

The catalog stores a base price per product and converts to the selected display
currency in real time. The checkout routes create an OpenNode charge using the
product's configured base price and return the BOLT11 invoice for the browser
paywall UI.

## Paywall UX

Each product page now exposes two payment paths:

- a browser-side paywall that generates an OpenNode charge, shows a QR code, polls for settlement, and unlocks the download button on-page
- an optional hosted checkout page supplied by OpenNode

The paid PDF download remains private until the invoice is verified as settled.

## OpenNode Webhook

The app exposes a verified webhook at:

- `/api/payment-webhook`

OpenNode sends `application/x-www-form-urlencoded` charge events there. The
route validates `hashed_order` with your API key before accepting the event.
